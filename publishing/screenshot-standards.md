# VIRON Method — Screenshot Standards
Version: 1.0  
Status: Production Enforcement Document  

Purpose:
This file defines strict standards for capturing, editing, and inserting screenshots into the VIRON manual.

Screenshots are tactical tools — not decorative elements.

---

# 1. Demo Environment Requirements

## 1.1 Dedicated macOS User

All screenshots must be taken from a dedicated macOS account.

Required configuration:

User Name: VIRON Demo  
Clean Apple ID  
No personal data  

Never use a personal production account.

---

## 1.2 System Cleanliness Rules

Before capturing:

- Disable notifications
- Remove unnecessary Dock icons
- Clear Desktop files
- Use neutral light-gray wallpaper
- Standardize window size

Consistency builds authority.

---

# 2. When to Use Screenshots

Screenshots are allowed only when:

- Teaching exact navigation path
- Demonstrating specific macOS configuration
- Showing UI that cannot be abstracted safely

Screenshots are NOT used for:
- Conceptual explanation
- System architecture
- Folder structure concepts

Use mockups instead.

---

# 3. Formatting Rules

## 3.1 Color Policy

- Convert to grayscale
- Remove color highlights
- No red arrows
- No bright annotation shapes

Use black callout numbers only.

---

## 3.2 Cropping Rules

- Tight crop around relevant area
- No excessive blank margins
- Remove irrelevant UI clutter
- Avoid full desktop captures

---

## 3.3 Size & Consistency

- Maintain consistent window width
- Maintain consistent zoom level
- Maintain consistent UI scaling
- Keep consistent margins when placed in document

---

# 4. Annotation Standards

If annotation is required:

- Use simple numbered circles (black outline)
- Provide explanation below image
- Do not write over UI elements

Never draw freehand arrows.

---

# 5. Naming Conventions

Screenshots must follow strict naming:

SC-[ChapterNumber]-[Description].png

Examples:

SC-05-Share-To-Reminders.png  
SC-02-Create-New-Folder.png  
SC-06-Reminder-Completed-Trigger.png  

All screenshots stored in:

/assets/screenshots/

---

# 6. Screenshot vs Mockup Decision Tree

If the image explains:

Conceptual structure → Mockup  
System architecture → Diagram  
Navigation path → Screenshot  
Precise UI interaction → Screenshot  

Default to mockup unless exact UI clarity is required.

---

# 7. Version Control

If macOS version changes:

- Review all screenshots
- Replace outdated UI references
- Update version notation in caption if needed

Screenshots should not lock the book to a single macOS release.

---

# 8. Privacy & Legal Safety

Never include:

- Real email addresses
- Real file paths
- Real client names
- Real calendar events
- Apple ID information

Screenshots must be safe for public publication.

---

# 9. Export Rules

- Save as PNG (high resolution)
- Avoid JPEG compression artifacts
- Insert into Word at 100% scale
- Do not resize disproportionately

---

# 10. Final Production Checklist

Before publication:

☐ All screenshots grayscale  
☐ Consistent margins  
☐ No UI clutter  
☐ No personal data  
☐ Naming convention followed  
☐ Stored in correct directory  
☐ Referenced in Diagram Registry  

---

End of Screenshot Standards
